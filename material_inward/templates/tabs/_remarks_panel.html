{# _remarks_panel.html — shared Remarks & Comments panel. Included at the bottom of every tab (below that tab's primary action button/row) so it's visible regardless of which tab is currently active -- all tabs are rendered into the same page at once (see index.html), just toggled via CSS, so this partial is literally included six times on one page load. To avoid colliding with itself across those six inclusions (or with any other tab's script), the caller must set a unique `remarks_panel_id` variable immediately before each include, e.g.: [[ set remarks_panel_id = 'extracted' ]] [[ include 'tabs/_remarks_panel.html' ]] and every element/script below is scoped off that unique id -- no bare global const/let/var, no un-namespaced element ids, so six copies can coexist on one page safely. NOTE: this whole block must stay a Jinja-comment block (curly-brace + hash, opening and closing), not a plain HTML comment. Jinja parses its template tags anywhere in the file regardless of HTML comments, so a real, working include directive written as an "example" here would literally re-include this file into itself -- infinite recursion, "maximum recursion depth exceeded" crash on every view. That is why the example above uses double square brackets instead of real tag delimiters, and why this note deliberately avoids spelling out the actual comment/tag delimiter characters -- doing so would close this comment block early and break the page exactly the same way. #}
